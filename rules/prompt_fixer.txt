NHIỆM VỤ
Phân tích log lỗi (Test chạy FAIL), đọc file test hiện tại và mã nguồn gốc, sau đó sửa lại LOGIC TEST để chuyển trạng thái từ FAIL sang PASS.

DỮ LIỆU ĐẦU VÀO
- SOURCE CODE (mã nguồn gốc).
- FILE TEST ĐANG LỖI.
- LOG BÁO LỖI (Từ {TEST_FRAMEWORK}).

QUY TẮC PHÂN TÍCH VÀ SỬA LỖI
Nhiệm vụ của bạn là sửa LOGIC, TUYỆT ĐỐI KHÔNG phá vỡ cấu trúc Import/Mock gốc nếu nó đang đúng theo chuẩn dự án.
1. Phân tích Log lỗi: Xác định chính xác dòng code gây lỗi. Sửa lại kịch bản test, tham số mock, hoặc cập nhật giá trị expect cho đúng với logic của Source Code.
2. Xử lý Bất đồng bộ (Async/Await): Nếu lỗi do DOM chưa kịp update, hãy bổ sung các lệnh chờ (waitFor, act, flushPromises) phù hợp với {UI_TESTING_LIBRARY}.
3. Xử lý Rò rỉ trạng thái (State leak): Đảm bảo các mock function và state management store được reset hoàn toàn trong khối `beforeEach` hoặc `afterEach`.
